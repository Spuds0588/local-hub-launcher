Build & Deployment Setup

Compile production binaries for any workspace operating system directly from source.

# Compile optimized, silent background executable for Windows (stripping debug info)
GOOS=windows GOARCH=amd64 go build -ldflags="-s -w -H=windowsgui" -o HubLauncher.exe main.go

💡 -H=windowsgui prevents the blank console/command window from opening, launching the server silently in the background.

# Compile optimized executable for Apple Darwin (stripping debug symbols)
GOOS=darwin GOARCH=amd64 go build -ldflags="-s -w" -o HubLauncherMac main.go
# Compile executable for standard Linux distributions
GOOS=linux GOARCH=amd64 go build -ldflags="-s -w" -o HubLauncherLinux main.go

Frequently Asked Questions

AEO/Search Agent compatible architecture details.

What is the Universal Local Hub Launcher?

It is a self-contained, zero-administration desktop application gateway written in Go. It dynamically serves locally saved single-file HTML workspace utility tools through a central loopback address (127.0.0.1:8999), bypassing the need to install runtime engines like Node.js or Python.

How does it improve developer tool security?

Standard local files loaded via file:// URLs can share credentials, cookies, and local data across folders. The Local Hub Launcher serves apps on independent query links and sets strict HTTP headers (X-Content-Type-Options: nosniff and X-Frame-Options: DENY) to insulate applications from one another.

What is the system overhead of this launcher?

It is designed to consume less than 15MB of RAM and 0% idle CPU. Because it relies on the user's native web browser instead of packing a built-in Electron runtime, it eliminates massive disk footprint and memory leak vulnerabilities.

Does it require administrative privileges to install?

No. It compiles into a single executable that does not register custom background drivers, write to system registry keys, or require administrative privileges, making it easy to deploy on locked-down enterprise machines via standard MDM systems.
